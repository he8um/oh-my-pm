# Playbook: Backlog Prioritization

Use this playbook to prioritize a backlog for a release or sprint.

## Steps

1. **Understand the goal** — What is this release trying to achieve?
2. **Review acceptance criteria** — Are criteria defined for each item? Flag items without criteria.
3. **Apply MoSCoW** — Must / Should / Could / Won't for this release.
4. **Check dependencies** — Does any Must have depend on a Won't have? Resolve conflicts.
5. **Validate with constraints** — Does the Must have list fit in the available time and team?
6. **Surface trade-offs** — What are you explicitly deferring and why?
7. **Document decisions** — Record the prioritization decisions in the decision log.

## Common mistakes

- Too many Must haves (everything can't be critical)
- Must haves that don't map to the stated goal
- Missing dependency analysis
- No explicit Won't haves (leads to scope creep later)
