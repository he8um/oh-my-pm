# Playbook: Retrospective

Use this playbook to run a structured retrospective after a sprint, milestone, or project.

## Steps

1. **Set the context** — What period or milestone does this retrospective cover?
2. **What went well** — What worked? What should we keep doing?
3. **What to improve** — What did not work? What should we do differently?
4. **Action items** — For each improvement, define one concrete action with an owner and date.
5. **Shoutouts** — Who deserves recognition?
6. **Follow up on previous retro actions** — Which were completed? Which were not?

## Output format

```
Period: [Sprint / Milestone / Project name]
Date: [Date]
Facilitator: [Name]

What went well:
- [Item]

What to improve:
- [Item]

Action items:
| Action | Owner | By |
|---|---|---|
| | | |

Shoutouts:
- [Name]: [What they did]
```

## Tips

- Keep it timeboxed: 60 minutes maximum.
- Focus on systemic issues, not individual blame.
- Every retro must end with at least one concrete action.
