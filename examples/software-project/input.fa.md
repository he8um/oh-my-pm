# مثال: پروژه نرم‌افزاری — ورودی (فارسی)

این یک مثال ساختگی است. هیچ شرکت واقعی یا داده خصوصی استفاده نشده است.

---

**پروژه:** یکپارچه‌سازی Payments API نسخه ۲
**تیم:** ۴ مهندس، ۱ QA، ۱ PM
**تایم‌لاین:** ۸ هفته
**هدف:** جایگزینی ارائه‌دهنده پرداخت قدیمی با API جدید برای کاهش نرخ شکست تراکنش‌ها

**وضعیت فعلی:**
- هفته ۳ از ۸
- یکپارچه‌سازی backend ۶۰٪ کامل
- محیط QA هنوز راه‌اندازی نشده
- اعتبارنامه‌های sandbox ارائه‌دهنده پرداخت تأخیر دارد — در انتظار vendor
- هیچ rollback plan مستندی وجود ندارد
- تاریخ لانچ به استیک‌هولدرهای کسب‌وکار اعلام شده

**بلاکرها:**
- اعتبارنامه‌های sandbox دریافت نشده (vendor یک هفته تأخیر دارد)
- راه‌اندازی محیط QA بلاک شده روی DevOps (تخمین ۳ روز)

**ریسک‌ها:**
- اگر sandbox بیشتر تأخیر کند، پنجره QA زیر حداقل فشرده می‌شود
- هیچ rollback plan در صورت شکست لانچ وجود ندارد

**استیک‌هولدرها:**
- CTO (اسپانسر اجرایی)
- VP Finance (مالک کسب‌وکار)
- لید مهندسی
- لید QA
