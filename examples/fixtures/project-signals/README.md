# Riverside Signals fixture

A fictional project used to exercise deterministic risk and next-task
extraction. All names, repositories, and content are invented for testing;
there is no real project, person, or private data here.

The fixture intentionally mixes:

- English and Persian risk headings, action headings, and inline markers
- blocker and dependency signals
- a checked (resolved) risk that must be excluded
- unchecked tasks and an action section
- Persian next steps
- a duplicate task line
- priority markers (`[P0]`, `Critical:`, `بحرانی:`)
- fenced code blocks whose contents are false positives
- ordinary prose containing the words "unblocked" and "riskless"
