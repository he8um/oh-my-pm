# مثال: پروژه دلیوری ترکیبی — خروجی (فارسی)

---

## تشخیص پروژه: طراحی مجدد onboarding مشتری

**وضعیت:** 🟡 زرد

مهندسی روی تأیید design بلاک است. نبود قرارداد API ریسک drift بین frontend/backend است.

## ریسک‌های اصلی

| ریسک | احتمال | تأثیر | اقدام کاهشی |
| --- | --- | --- | --- |
| قرارداد API نهایی نشده — drift بین frontend/backend | بالا | بالا | قرارداد API قبل از شروع مهندسی تعریف شود. tech lead مسئول شود. |
| تأخیر تأیید design تا بعد از بررسی استیک‌هولدرها | متوسط | بالا | تایم‌لاین بررسی design تأیید شود. در صورت عدم تکمیل قبل از kickoff مهندسی، اسکالیشن شود. |
| نبود rollback plan | بالا | متوسط | معیارهای rollback قبل از اولین ریلیز به production تعریف شود. |

## اقدامات فوری

| اقدام | مالک | تا |
| --- | --- | --- |
| نهایی کردن قرارداد API | tech lead | قبل از kickoff مهندسی |
| تأیید تایم‌لاین تأیید design | PM | امروز |
| تعریف معیارهای rollback | لید مهندسی | هفته ۲ |
| اولویت‌بندی بک‌لاگ با تیم | PM | قبل از بررسی استیک‌هولدرها |
