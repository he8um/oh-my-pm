# Philosophy

## Delivery leadership is a skill, not just a role

A Head of Delivery does not just manage tasks. They bring judgment: when to push, when to pause, how to prioritize under real constraints, how to communicate bad news clearly, and how to keep teams aligned without bureaucracy.

Oh My PM encodes this judgment as a reusable, installable agent layer.

## Agents need structure, not just instructions

Generic AI agents produce generic outputs. Oh My PM gives agents the structure they need to think like a delivery leader: intake before action, critical path before recommendation, risk before optimism.

## Token discipline is a delivery value

Padded outputs slow down delivery work. Oh My PM agents produce dense, structured, actionable content — the kind a real PM would produce under time pressure.

## Bilingual support is a real need

Many delivery teams operate across Persian and English. Persian business context with English technical identifiers is not a translation problem — it is a workflow reality. Oh My PM supports this natively.

## Open source by design

Oh My PM is MIT-licensed and tool-agnostic. It works with Claude, Cursor, Codex, ChatGPT, and any agent that reads markdown. No vendor lock-in. No proprietary format.

## MCP is optional and future

MCP integration is planned as an optional data access layer. It will not replace the core agent kit. It will add project tool connectivity for teams that want it.
