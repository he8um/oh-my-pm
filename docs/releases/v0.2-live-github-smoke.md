# OH MY PM — v0.2 Live GitHub Smoke

## Purpose

A controlled, read-only live-network validation of the actual installed `v0.2`
portable artifact against the real GitHub REST API, run before Release Candidate
preparation. This is a validation phase, not a feature phase: no product code,
tests, installer, bundle, CI, or version were changed. The smoke exercises the
installed CLI and the installed MCP server (never source imports or a test
harness) and verifies the bounded, sanitized read-only behavior end to end.

## Baseline

- UTC date: 2026-07-24
- Commit under test: `f16b2fd` — refactor: centralize GitHub limit constants
- Version: `0.2.0-alpha.0` (unchanged; unreleased alpha)
- Baseline gates green before the smoke: version consistency, structure,
  boundaries, `pnpm build`, `pnpm test` (2053), `pnpm validate`, `pnpm mcp:smoke`;
  worktree clean.

## Artifact Under Test

- Installed portable release bundle built via `tools/build-release-bundle.mjs
  --apply`, verified via `tools/check-release-bundle.mjs`, moved outside the
  build directory, then installed with `bin/oh-my-pm-install.mjs --apply` into a
  temporary `<temporary-prefix>`.
- Installed-state verified via `tools/check-release-install.mjs` (PASS).
- Source-bundle independence: the extracted bundle was deleted and the installed
  `oh-my-pm status` still reported `healthy` (PASS).
- OS / runtime: macOS (Darwin, arm64); local Node `v26`. (Release CI additionally
  exercises the installed artifact on Ubuntu and Windows with Node 20.)

## Safety Controls

- `OH_MY_PM_GITHUB_TOKEN` was the only accepted token source and was **absent**
  in this run; tokenless commands were invoked with `env -u OH_MY_PM_GITHUB_TOKEN`.
- No token was echoed, written to a file, hashed, placed in arguments/config, or
  committed. No `set -x`. No environment dumps.
- All temporary output was confined to a temp dir and deleted before staging.
- GET-only, single page, small limits (≤5). No writes, non-GET methods, GraphQL,
  retries, pagination, caching, or sync.

## API Compatibility

The live GitHub REST responses were accepted and normalized without error across
repository metadata, issues list, search, and a merged PR's issue detail, PR
detail, ordinary comments, reviews, and inline review comments. Canonical review
state mapped correctly (an `APPROVED` review surfaced as an approval handoff
decision). The fixed origin `https://api.github.com` and API version `2026-03-10`
declared by `providers doctor` matched the live surface in use.

## Rate-Limit Observations

No rate-limit exhaustion was encountered on the unauthenticated public endpoints
during this run (small limits, single page, no retries). No `OMP-P-4007`
(rate-limited) failure occurred. Preflight and smoke together made a small,
bounded number of GET requests.

## Data-Exposure Review

A scan of all temporary output found no `Authorization`/`Bearer`, no
`OH_MY_PM_GITHUB_TOKEN`, no `diff_hunk`/`commit_id`/`original_commit_id`/`node_id`,
no raw response headers, no raw resource API URLs, no absolute local paths in the
markdown outputs, and no stack traces. The only `api.github.com` occurrence was
the documented fixed origin in `providers doctor` output (allowed). No raw
third-party comment/review/review-comment body was emitted; handoff/summary
content came only from the primary PR item's own body and bounded state-derived
signals. No temporary output was committed.

## Defects Found

None. No product-behavior or data-boundary defect was observed.

## RC Readiness Decision

**READY FOR RC PREPARATION.** The mandatory tokenless CLI flow and the installed
MCP live workflows passed against the real GitHub API using the installed portable
artifact, with correct bounded, sanitized projections and correct negative-path
behavior, and no forbidden data exposure. The authenticated CLI smoke was NOT RUN
(token unavailable), which does not block RC. This decision does not release
`v0.2` and does not bump the version; the next action is `0.2.0-rc.1`
preparation as a separate, deliberate step.
