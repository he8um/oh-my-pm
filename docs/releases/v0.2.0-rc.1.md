# OH MY PM v0.2.0-rc.1

## Status

**Release candidate — prerelease, not stable.** This is the first `v0.2` release
candidate. As prepared, nothing is published: there is no `v0.2.0-rc.1` tag,
GitHub Release, or download yet, and the latest **stable** release remains
`v0.1.0`. When published, `v0.2.0-rc.1` is marked as a GitHub *prerelease* and is
never marked "latest". Publication is a separate, manually gated step — see
[publishing-v0.2.0-rc.1.md](publishing-v0.2.0-rc.1.md).

## What this release candidate is

A candidate build of the `v0.2` line for validation before a stable `v0.2.0`.
It is a self-contained, versioned portable artifact that runs on Node.js 20+
with no Rust toolchain, pnpm, or repository checkout required. It bundles the
compiled packages, the real prebuilt Rust/WASM Kernel, the CLI, and the stdio
MCP server. There is no npm or crates.io registry publication; distribution is
by verifiable release archives only.

## Highlights

`v0.2` adds an explicit, read-only GitHub provider and richer deterministic
project intelligence on top of the local Markdown workflows from `v0.1.0`, plus
a portable preview-first installer with cross-platform verification.

### GitHub provider

- Strictly read-only, explicitly opt-in access to `api.github.com`. `GET`-only,
  fixed origin, fixed REST API version, single page per request, optional token.
- Source selection: `overview`, `repository`, `issues`, `pull-requests`, a single
  `item` (issue/PR with type auto-detection), and repository-scoped `search`,
  each with `open`/`closed`/`all` state and a search `kind`.
- Bounded, default-disabled discussion context for the `item` source: ordinary
  issue/PR conversation comments; and, only when the item is a pull request,
  review submissions and inline review comments. Each is a single extra page with
  small bounds.
- Provider configuration (`providers.json`) with optional GitHub defaults and an
  offline `providers status` / explicitly confirmed `providers doctor` diagnostic.

### CLI

- `oh-my-pm github <brief|risks|next|handoff>` over the selected source, plus the
  local `brief|risks|next|handoff` workflows and `providers status|doctor`.
- Bounded discussion flags: `--include-comments`/`--comment-limit`,
  `--include-reviews`/`--review-limit`, `--include-review-comments`/
  `--review-comment-limit` (item source; reviews are pull-request-only).

### MCP

- stdio transport only, with exactly **ten** tools (four local project tools,
  four GitHub workflow tools, `provider_status`, `github_provider_diagnostics`).
- Sanitized public projections only: bounded review/comment metadata (identity,
  state, file provenance) — never raw bodies, diff hunks, or commit identifiers,
  and never a raw provider object, token, header, config path, runtime trace, or
  planner graph.

### Deterministic project intelligence

- Source-aware, line-level risk / next-task / decision extraction for Markdown
  and GitHub context, English and Persian headings and markers, exact GitHub
  label/status rules, overdue inference from an injected timestamp, and bounded
  risk/task limits. Rule-based only — no LLM, embedding, or fuzzy scorer.

### Portable installation

- Every archive ships a preview-first installer at `bin/oh-my-pm-install.mjs`.
  Preview is the default; `--apply` is required to write; installation targets an
  explicit `--prefix`. The installed prefix is source-independent and relocatable.

## Security and privacy guarantees

- Local Markdown workflows are fully offline and read no token.
- GitHub access is explicit, read-only, `GET`-only, single fixed origin
  (`https://api.github.com`), fixed API version, single page, optional
  `OH_MY_PM_GITHUB_TOKEN` (never accepted as a CLI/MCP argument, never printed or
  persisted). No writes, GraphQL, retries, pagination, caching, or sync.
- No raw comment/review/review-comment bodies, diff hunks, or commit identifiers
  are exposed through the MCP projections.
- The installer performs no `PATH`, shell-profile, project, or MCP-client-config
  edits; it writes only under the explicit prefix.
- No telemetry, no project-file writes, no context upload. All packages private.

## Known limitations

- Discussion context is bounded to a single page per endpoint (comments ≤ 50;
  reviews and inline review comments ≤ 20 each) and disabled by default.
- Review options apply only to a pull-request `item`; selecting an issue with a
  review option is a controlled, sanitized failure.
- The bundled Kernel is a prebuilt Rust/WASM artifact for Node.js 20+.

## Deferred features

The following are intentionally **not** part of `v0.2` and are deferred to a
later, separately approved phase:

- timeline events
- review-thread graph / resolution
- reactions
- commits / files / diffs
- provider aliases / profiles
- controlled write-back
- HTTP MCP transport
- telemetry, dashboard, registry publishing

## Installation

Once published, download the archive for your platform from the GitHub
prerelease, extract it, preview, then apply into an explicit prefix:

```bash
tar -xzf oh-my-pm-v0.2.0-rc.1.tar.gz          # or: unzip oh-my-pm-v0.2.0-rc.1.zip
# Preview (writes nothing):
node ./oh-my-pm-v0.2.0-rc.1/bin/oh-my-pm-install.mjs --prefix "$HOME/.local"
# Apply (writes only under the prefix):
node ./oh-my-pm-v0.2.0-rc.1/bin/oh-my-pm-install.mjs --prefix "$HOME/.local" --apply
```

Each archive extracts to exactly one top-level directory: `oh-my-pm-v0.2.0-rc.1/`.

## Upgrade notes from v0.1.0

- `v0.2` is backward-compatible for the local Markdown workflows. The bundle
  name, archive layout, and preview-first installer follow the same shape as
  `v0.1.0`, with the version string updated to `0.2.0-rc.1`.
- The GitHub provider, provider configuration/diagnostics, and the six additional
  MCP tools (GitHub workflows + diagnostics) are new and strictly opt-in; existing
  offline usage is unchanged.
- As a release candidate, install it alongside — not over — a stable `v0.1.0`
  installation by choosing a distinct `--prefix` if you want both.

## Release-candidate validation

Before this candidate was prepared, the following passed on `main`:

- full contract, structure, boundary, public-surface, and version-consistency
  gates; all TypeScript and Rust builds; `pnpm test`; `pnpm validate`;
  `pnpm mcp:smoke`; `cargo test`; `cargo clippy -D warnings`; the
  `wasm32-unknown-unknown` Kernel build.
- cross-platform release-install CI parity: the Windows job (like the POSIX job)
  proves the installed artifact survives deleting the extracted source bundle and
  moving the complete installed prefix, re-verifying the installed CLI and MCP.
- a controlled, read-only live GitHub smoke through the **installed** portable
  artifact: the tokenless provider/CLI flows and the installed MCP live workflows
  passed with bounded, sanitized projections. The authenticated smoke was **not
  run** because no token was available; the tokenless and installed-MCP checks
  carried the acceptance. See [v0.2-live-github-smoke.md](v0.2-live-github-smoke.md).
