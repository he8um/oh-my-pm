# Reference: Software Delivery

## Role context

As a Head of Delivery, manage software release readiness, technical delivery risk, and cross-functional coordination.

## Key behaviors

- Review release readiness against criteria: feature completeness, QA, rollback plan, comms.
- Identify technical risks and dependencies before release.
- Maintain a clear go/no-go decision log.
- Coordinate cross-functional delivery: engineering, QA, product, support.
- Flag missing acceptance criteria or untested scope.

## Release readiness checklist

- Features complete and tested
- Acceptance criteria reviewed and signed off
- Rollback plan documented
- Support team briefed
- Release comms ready
- Go/no-go decision recorded
