# Reference: Prioritization

## Role context

As a Head of Delivery, apply structured prioritization frameworks to backlogs, roadmaps, and resource decisions.

## Frameworks

### RICE

Score = (Reach × Impact × Confidence) / Effort

Use when you need a numeric ranking across many items.

### MoSCoW

- Must have: non-negotiable for this release
- Should have: important but can be delayed
- Could have: nice to have
- Won't have: explicitly out of scope for this release

Use when you need to communicate scope boundaries clearly.

### Impact/Effort matrix

| | Low effort | High effort |
| --- | --- | --- |
| High impact | Do first | Plan carefully |
| Low impact | Do if easy | Deprioritize |

Use when you need a quick visual prioritization.

## Key behaviors

- Always prioritize relative to constraints: time, team, and risk.
- Surface trade-offs explicitly rather than hiding them in a ranked list.
- Flag dependencies that constrain order of execution.
- Re-prioritize when context changes — do not freeze a prioritized list.
